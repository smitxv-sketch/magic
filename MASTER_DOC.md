# Мастер-файл проекта (Архитектура и Документация)

Этот файл служит отправной точкой для ИИ и разработчиков. Здесь описана структура проекта, чтобы не тратить токены на чтение всех файлов.

## 📂 Структура проекта

*   **`/src/App.tsx`** — Корневой компонент приложения.
*   **`/src/components/Layout/AppShell.tsx`** — Основная оболочка приложения, роутинг между режимами (Презентация, Студия, Действия, Кейсы, ИТ-Хаб).
*   **`/src/store/`** — Стейт-менеджмент (Zustand).
    *   `appStore.ts` — Глобальное состояние приложения (режимы, модалки, ключи API).
    *   `tenantStore.ts` — Состояние текущего тенанта (клиента), загрузка конфигурации и контента.
*   **`/src/data/`** — Данные и конфигурации.
    *   `presentation_content.json` — Данные для лендинга/презентации (тексты, картинки).
    *   `studio_scenarios.ts` — Сценарии и промпты для Студии.
*   **`/src/components/presentation/`** — Компоненты лендинга (Презентации).
    *   `PresentationView.tsx` — Главный файл-сборщик лендинга.
    *   `blocks/` — Отдельные экраны лендинга (HeroIntroSection, ProblemChaosSection, SolutionRoutingSection, ShadowModeIntroSection, DemoShadowAuditSection, DemoSecretarySection, DemoLawyerSection, ExoskeletonConceptSection, ToolOrchestratorSection, MetricsRoiSection, ItArchitectureSection, FutureAgentsSection, CtaPilotSection). Каждый экран независим для удобного рефакторинга.
*   **`/src/components/Studio/`** — Компоненты режима "Студия" (конструктор промптов, настройка AI).
*   **`/src/components/Player/`** — Компоненты режима "Плеер" (демонстрация работы AI).
*   **`/docs/`** — Папка со старыми ТЗ и исторической документацией (`tz.md`, `doc_1.md` и т.д.).

## 🛠 Принципы разработки
1.  **Zero Hardcode (по возможности):** Тексты и конфигурации выносятся в `/src/data/` или стейт, но для лендинга мы используем независимые секции для гибкости верстки.
2.  **SSOT (Single Source of Truth):** Единый источник истины для данных.
3.  **Разделение ответственности:** Каждый экран лендинга — это отдельный файл в `/src/components/presentation/blocks/`, а `PresentationView.tsx` только собирает их вместе. Это позволяет безопасно рефакторить отдельные блоки.
