Ты приходишь не просить бюджет на ИТ-игрушку, а предлагаешь радикально сократить операционные издержки своего же подразделения и смежных отделов, используя современные инструменты. Это язык бизнеса, который понимают и одобряют.

Чтобы избежать техдолга и хардкода, мы спроектируем `PresentationView.tsx` не как статичную страницу, а как **динамический рендерер (Layout Engine)**. Он будет брать массив объектов из JSON и, в зависимости от типа объекта (например, `hero`, `split-view`, `bento`), подставлять нужный UI-компонент.

### 2. Типизация данных (Zero Tech Debt Architecture)

Чтобы разработчик сверстал это чисто, мы сразу задаем TypeScript-интерфейсы. Это гарантия того, что JSON будет строго соответствовать дизайн-системе. Передай это разработчику вместе со следующим шагом.

```typescript
// types/presentation.ts

export type BlockType = 
  | 'hero' 
  | 'split-left' 
  | 'split-right' 
  | 'bento' 
  | 'it-summary' 
  | 'vision-agents' 
  | 'cta';

export interface PresentationBlock {
  id: string;
  type: BlockType;
  title: string;
  subtitle?: string;
  content?: string | string[]; // Текст или список буллитов
  imageUrl?: string; // Ссылка на картинку из /public/images/
  metrics?: Array<{ label: string; value: string }>; // Для Bento-блока
  actions?: Array<{ label: string; primary: boolean; href?: string }>; // Кнопки
}

export interface PresentationData {
  meta: {
    title: string;
    version: string;
  };
  blocks: PresentationBlock[];
}

```

# ТЕХНИЧЕСКОЕ ЗАДАНИЕ: Движок презентации (Kaizen Presentation Engine)

> **КОНТЕКСТ ДЛЯ AI-АГЕНТА:** > Необходимо разработать новый корневой экран приложения — `PresentationView.tsx`. Это не просто статичная страница, а динамический **Layout Engine**. Весь контент (тексты, ссылки на картинки, метрики) загружается из локального JSON-файла.
> Цель экрана — интерактивная презентация Кайдзен-инициативы для проектного комитета компании "Уралэнергосбыт". Дизайн должен соответствовать премиальному Enterprise-сегменту (Tailwind CSS, Glassmorphism, плавная анимация при скролле через Framer Motion).

## ШАГ 1: Типизация данных (Strict TypeScript Interfaces)

Создайте файл `src/types/presentation.ts`. Эта схема гарантирует отсутствие технического долга и строгую типизацию пропсов.

```typescript
export type BlockLayoutType = 
  | 'hero' 
  | 'split-right' // Картинка справа, текст слева
  | 'split-left'  // Картинка слева, текст справа
  | 'bento'       // Сетка карточек с метриками
  | 'it-summary'  // Блок ИТ-архитектуры
  | 'vision'      // Блок будущего (Агенты)
  | 'cta';        // Призыв к действию (Пилот)

export interface ContentItem {
  icon?: string; // Название иконки из lucide-react (например, 'FileText', 'Shield')
  title?: string;
  text: string;
}

export interface MetricItem {
  value: string;
  label: string;
  trend?: 'up' | 'down' | 'neutral';
}

export interface PresentationBlock {
  id: string;
  layout: BlockLayoutType;
  badge?: string; // Маленький лейбл над заголовком (например, "Кайдзен-инициатива")
  title: string;
  subtitle?: string;
  content?: ContentItem[]; 
  imageUrl?: string; // Путь к локальной картинке, например "/images/hero.png"
  metrics?: MetricItem[];
  primaryAction?: { label: string; actionId: string }; // actionId для обработки кликов (например, 'open-player')
}

export interface PresentationData {
  meta: {
    company: string;
    department: string;
    version: string;
  };
  blocks: PresentationBlock[];
}

```

## ШАГ 3: Архитектура Компонента (PresentationEngine)

Создайте компонент `src/components/presentation/PresentationView.tsx`.

**Требования к рендеру:**

1. Использовать `framer-motion` для анимации появления секций (`whileInView={{ opacity: 1, y: 0 }}`, эффект всплытия).
2. Написать функцию `renderBlock(block: PresentationBlock)`, которая использует оператор `switch (block.layout)` для возврата нужного куска верстки.
3. **Glassmorphism:** Использовать классы Tailwind вроде `bg-white/70 backdrop-blur-xl border border-white/40 shadow-xl rounded-3xl` для карточек и изображений.
4. **Типографика:** Для заголовков H2 использовать `text-4xl md:text-5xl font-extrabold tracking-tight text-slate-900`.
5. **Обработка кнопок:** Если кнопка имеет `actionId === 'open-player'`, она должна менять стейт приложения (Zustand) на переключение вкладки Плеера.

## ШАГ 4: Интеграция с навигацией и UX сохранения позиции

В `AppShell.tsx` или в Zustand-хранилище необходимо реализовать запоминание позиции скролла:

* Когда юзер кликает "Смотреть живое демо", он переходит на вкладку Плеера.
* На других вкладках внизу экрана должна плавать кнопка `← Вернуться к презентации`.
* При нажатии на неё, `PresentationView` монтируется и немедленно делает `window.scrollTo(0, savedScrollPosition)`, возвращая комиссию ровно на тот слайд, откуда они ушли смотреть демо.
