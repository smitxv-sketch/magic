### Патч ТЗ №3. Интерактивные Симуляторы: «Магия ИИ» и Тайминги

**Цель:** Оживить компоненты симуляции (`PlayerWorkspace` / `AI_Node`). Добавить искусственные задержки, поочередный вывод проверок, эффект печатающегося текста и кнопку перезапуска демо для спикера.

**Задачи для Front-end разработчика:**

**1. Последовательный рендер проверок (AI Thinking Delay):**
В компоненте, который рендерит `ai_node` и его массив `boolean_checks_config`, необходимо отказаться от мгновенного `map()`.

* *Требование:* Пункты чеклиста должны появляться строго по одному.
* *Реализация:* Использовать хук `useEffect` и таймеры. Когда активен `ai_node`, сначала показывается пульсирующий лоадер (статус: *«Анализ текста...»*). Через ~1.2 секунды появляется первая проверка. Еще через ~800мс — вторая. И так далее.

**2. Эффект печатающегося текста (Typewriter) для результатов:**
Поля `result_text` (например, *"ОШИБКА: 60% + 50% = 110%"*) не должны появляться целиком.

* *Требование:* Использовать библиотеку (например, `framer-motion` или кастомный хук) для посимвольного вывода текста ответа ИИ, чтобы это выглядело как генерация токенов в реальном времени (как в ChatGPT).

**3. Драматический акцент на ошибках (Visual Impact):**
Когда рендерится пункт с `is_passed: false` (наши кабальные пени или 110%):

* Экран/карточка должна сделать легкий визуальный "удар" (вспышка цвета `ues-amber` или красного).
* Процесс симуляции (если он шел автоматически) должен эффектно "споткнуться" — показать статус "ОБНАРУЖЕН КРИТИЧЕСКИЙ РИСК. ДАЛЬНЕЙШИЙ МАРШРУТ ЗАБЛОКИРОВАН".

**4. Внедрение кнопки «Рестарт Демо» (Replayability):**
На презентации часто просят: *"Ой, я отвлекся, покажите еще раз, как он нашел ошибку"*.

* *Требование:* В правом верхнем углу блока `InteractiveDemoBlock` (или самого `PlayerWorkspace`) добавить иконку повтора (`RefreshCw` из `lucide-react`).
* *Логика:* При клике стейт симуляции (`currentNodeIndex`, заполненные проверки) полностью сбрасывается в 0, и демо можно запустить заново по кнопке "Play".

*Пример логики для последовательного вывода (псевдокод для разработчика):*

```tsx
const AiChecklist = ({ checks }) => {
  const [visibleCount, setVisibleCount] = useState(0);

  useEffect(() => {
    if (visibleCount < checks.length) {
      const timer = setTimeout(() => {
        setVisibleCount(prev => prev + 1);
      }, 1200); // 1.2 секунды на "раздумья" перед каждым пунктом
      return () => clearTimeout(timer);
    }
  }, [visibleCount, checks.length]);

  return (
    <div className="space-y-3">
      {checks.slice(0, visibleCount).map((check, i) => (
        <CheckItem key={i} check={check} typewriterEffect={true} />
      ))}
      {visibleCount < checks.length && <PulsingLoader text="Анализ нейросетью..." />}
    </div>
  );
};

```

Ты попал в самую десятку. Это классическая проблема внедрения инноваций: бизнес боится, что ИИ начнет принимать решения и «наломает дров» в первый же день. То, что ты предлагаешь — в ИТ называется **«Теневое развертывание» (Shadow Mode)**.

Для твоего Амбассадора (Начальника УД) это идеальная метафора: *"Мы не даем ИИ руль. Мы сажаем его на пассажирское сиденье с блокнотом. Он просто смотрит, как работают ваши люди, и пишет отчет, где мы теряем время"*.

Это полностью снимает страх. Мы показываем, что первый этап — это безопасный аудит, «интеллектуальный градусник».
